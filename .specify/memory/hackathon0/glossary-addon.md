# GLOSSARY ADDON — HACKATHON 0

## Purpose

This document defines key terms used in the Hackathon 0 system to ensure consistent understanding.

---

## Terms

### Task
A structured representation of incoming work generated from an intake source.

### Plan
A structured breakdown of actions derived from a task, prepared for human approval.

### Approval
A human decision that allows or rejects execution of a plan.

### Execution
The process of performing approved actions in the system.

### Logs
Append-only records of system behavior and execution events.

### Artifacts
Generated outputs such as approval files, posts, drafts, or records created during execution.

### HITL (Human-in-the-Loop)
A control mechanism requiring human approval before execution.

### Task ID
A unique identifier that remains consistent across task, plan, approval, and execution stages.

### Workflow
The lifecycle of a task from intake to completion.

### Gold Layer
The layer proving real system execution with logs and artifacts.

### Platinum Layer
The layer enhancing visibility, control, and system presentation.

### Updates Layer
The Platinum flow managing Incoming, Claimed, and Processed updates.

### Claim-by-Move
The action of moving a file from Incoming to Claimed to establish ownership.

### Deterministic System
A system where the same input produces predictable and traceable output.

---

## Final Note

All terms must be interpreted based on actual system behavior, not assumptions or external definitions.
